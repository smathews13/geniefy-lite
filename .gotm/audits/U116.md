### LOW severity
- **Residual envelope-vs-field tension in the table prompt.** The shared `_GROUNDING_RULES` line still ends `"… Return ONLY the JSON object, no prose."` while the new CRITICAL clause demands the `proposed_comment` *field value* be prose. Cosmetic only: the "no prose" phrasing could be reworded to "no extra prose outside the JSON" for zero-ambiguity. Not blocking.

## Detailed check results

1. **Prompt demands PROSE / forbids JSON dump — unambiguous.** PASS. A model reading this writes prose.

2. **Richer aspects still requested — no U104 regression.** PASS. Both `list(self.template.table_required)` and `list(self.template.table_recommended)` are still interpolated (reasoner.py:166–167). The fix changed FORMAT (now "cover these aspects in prose") not content coverage — the wording shifted from "fill these fields" to "naturally cover these aspects", which is the correct semantic for prose.

3. **Tag rule + reuse rule still present.** PASS. `_tag_rule('table')` and `_REUSE_RULE` both appear in the rendered table prompt (reasoner.py:170). `comment_library`, "PREFER reusing", and the free-form tag examples all present.

4. **JSON shape line intact.** PASS. The shape line declares `{"proposed_comment": str (the PROSE comment), "rationale": str, "tags": [str], "evidence_refs": [str], "self_confidence": number, "open_question": str|null}` (reasoner.py:171–172) — `proposed_comment` typed as a (prose) string, and tags/rationale/evidence_refs/self_confidence/open_question all retained.
