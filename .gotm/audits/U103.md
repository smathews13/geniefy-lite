In the UPDATE, `status` is the penultimate param and `id` (`existing[0]`) is the last/WHERE param regardless of the bump fragment (`:265‚Äì270`) ‚Äî verified against tests `:345`/`:357`. A brand-new entry hardcodes `usage_count ‚Ä¶ 1` in the INSERT VALUES (`:278`) and carries `status` as the last param (`:282`, test `:365`).

**Reuse query (¬ßA4).** `list_library_for_reuse` short-circuits to `[]` with no DB call when keys are empty (`:340‚Äì342`, test `:403` asserts `conn.executed == []`).
