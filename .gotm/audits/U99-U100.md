## U100 findings

Code matches spec on every point:

- `state.py`: `Question.suggested_answer: str | None = None` (line 293) with
  symmetric `to_dict` (line 302) and `from_dict` (line 313) — round-trips.
- `reasoner.py` `suggest_answers(state, questions)` (lines 99-137):
  - **One batched `complete_json` call** for all pending questions (lines 124-128).
  - **Truly best-effort** — `LLMError` is caught and returns (line 129-130);
    a non-dict response returns silently (lines 131-132); blank/whitespace
    answers are ignored via `ans.strip()` (lines 134-137). **It cannot raise
    on the model/parse path** and never blocks the run. (Confirmed by
    `test_suggest_answers_best_effort_on_bad_json`.)
  - Only fills questions whose `suggested_answer` is still `None` (line 104) —
    idempotent across resume.
- `orchestrator.py` `_gate` (lines 137-155): calls `suggest_answers` **only**
  when `interactive and questions` (line 142), behind a `getattr` guard
  (line 145, safe if a fake reasoner lacks the method) **and** a `try/except
  Exception: pass` (lines 148-151). Non-interactive and no-question paths skip
  it entirely and fall through to `ready_for_review` / `needs_input` exactly as
  before — guard is safe. Wrapped in a `suggest_answers` tracer span.
- `QuestionsPanel.tsx`: pre-fills each box via `valueFor` =
  `text[q.id] ?? q.suggested_answer ?? ''` (line 13) and shows the
  "✨ suggested" badge when the value is an unedited suggestion (lines 14, 49-53);
  editing the box clears the suggested state (badge disappears). Editable.
- `types.ts`: `Question.suggested_answer: string | null` (line 51).
- Tests present and meaningful: `test_suggest_answers_populates_from_llm`
  (populate + blank-ignored, lines 285-297) and
  `test_suggest_answers_best_effort_on_bad_json` (no-raise, line is set to
