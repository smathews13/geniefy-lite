## Findings

### HIGH severity
- (none)

The four HIGH-risk axes are all clean:
- **No UC write on approve.** `approve_high_confidence` (api.py:227–247) only sets `draft.status =
  APPROVED` + a best-effort library upsert + `store.save`; it never touches the `applier`, never
  flips `apply_status`, and writes no `audit_log` row — exactly the reconciled D59/U154 MED1/MED2
  semantics (approve ≠ apply; audit is apply-time/OBO). The `/approve-high-confidence` route
  (api.py:608–616) runs as the app SP (`_actor()`), not the OBO user, and has no `RequestIdentity`
  dependency — correct, since no UC write occurs.
- **Correct session targeted.** `useApproveTableInRun(s.session_id, runId)` (SchemaRunView.tsx:23)
  POSTs to `/api/sessions/${sessionId}/approve-high-confidence` (hooks.ts:162–168) for THAT row's
  session and invalidates `['schema-run', runId]` on success → the row refreshes. Per-row `s.session_id`
  comes from `list_sessions` (store.py:250), the same key `get_schema_run` uses to attach the summary
  (api.py:395) — consistent throughout.
- **No nested buttons.** `SchemaTableRow` renders a `<li>` (SchemaRunView.tsx:27) containing two
  SIBLING `<button>`s — the open button (28–49) and the Approve button (50–62). The Approve button is
  NOT inside the open button. Valid HTML; restructured from U112's single open-button row.
- **Build / lint / pytest all green.** `npm run build` → tsc `--noEmit` + vite both clean (0 errors,
  bundle built). `npm run lint` → eslint 0 problems. `PYTHONPATH=src .venv/bin/python -m pytest -q` →
  **392 passed, 1 skipped**. `tests/test_api.py:426` asserts `approvable == 3` (table+a+d are
  draft & ≥0.75), wired to `_high_confidence_targets`.
- **Type shape matches.** TS `ConfidenceSummary` (types.ts:103–110) = `overall|null · review_ready ·
  needs_input · low · approvable · weakest{target,confidence}|null`, exactly the server dict
  (api.py:120–127). `confidence_summary?: ConfidenceSummary | null` on `SessionSummary` (types.ts:122)
  matches the server attaching `None` when a session can't load (api.py:397).

  `title` tooltip (SchemaRunView.tsx:39), so a bare number is never shown without the breakdown
