## Check 1 ŌĆö D38 tests bite, with correct param indices

**Param-index verification (the load-bearing concern).** store.py:73-76 INSERT column order:
`(id, uc_catalog, uc_schema, uc_table, mode, status, template_id, config, session_state,
mlflow_run_id, created_by)`. The params tuple (store.py:85-87) is positionally aligned:

| idx | column        | param expr                              |
|-----|---------------|-----------------------------------------|
| 0   | id            | `sid`                                   |
| 1   | uc_catalog    | `cat`                                   |
| 2   | uc_schema     | `sch`                                   |
| 3   | uc_table      | `tbl`                                   |
| 4   | mode          | `state.config.mode.value`               |
| 5   | status        | `state.session_status.value`            |
| **6** | **template_id** | **`_uuid_or_none(state.template_id)`** |
| 7   | config        | `json.dumps(state.config.to_dict())`    |
| **8** | **session_state** | **`json.dumps(state.to_dict())`** (the jsonb snapshot) |
| 9   | mlflow_run_id | `state.mlflow_run_id`                    |
| 10  | created_by    | `created_by`                             |

So **index 6 = template_id column** and **index 8 = session_state snapshot** ŌĆö exactly the
indices the new tests assert on. The indices are NOT vacuous/misleading. (Consistent with the
existing `test_save_writes_session_snapshot_and_normalized_fields`, which already keys off
`params[8]` for the snapshot and `params[5]`/`params[10]` for status/created_by.)

- `test_uuid_or_none_writes_uuid_or_null` (test:183-189) covers all three D38 cases: uuid ŌåÆ
  canonical pass-through (`_uuid_or_none(u) == u`), name `"default"` ŌåÆ `None`, and `None`/`""`
  ŌåÆ `None`. Matches D38 verbatim ("set only if it parses as a uuid, else NULL").
- `test_save_writes_template_name_as_null_but_keeps_it_in_snapshot` (test:192-199) asserts
  `params[6] is None` (template_id FK column NULL) AND `json.loads(params[8])["template_id"]
  == "default"` (name preserved in the jsonb snapshot) ŌĆö the end-to-end D38 contract.

**Mutation probe ŌĆö D38 regression.** Replaced `_uuid_or_none`'s body with `return value`
(the precise regression the U48 MED warned about: writing the raw name to the FK column). Both
D38 tests failed (`test_uuid_or_none_writes_uuid_or_null`; and the save test at line 198, the
`params[6] is None` assert). The tests bite. The U48 MED (D38 lacked a *direct* test) is closed.
