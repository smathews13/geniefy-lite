## Check-by-check

1. **Tags model (state.py).** `tags: list[str] = field(default_factory=list)` is present on **both** `TableDraft` (L195) and `ColumnDraft` (L224). `_draft_common_to_dict` emits `"tags": list(getattr(o, "tags", []) or [])` (L261) and `_draft_common_from_dict` reads `o.tags = list(d.get("tags") or [])` (L277) — so tags round-trip through both drafts, default to `[]`, and tolerate a missing key (old snapshots) on both the to_dict (getattr guard) and from_dict (`.get(...) or []`) sides. `test_draft_tags_round_trip` (test_state.py L235) asserts round-trip on both, default empty, and missing-key tolerance. PASS.

2. **Reasoner tag extraction + free-form (not a locked taxonomy).** `_apply_common` (reasoner.py L304-311) normalizes: `str(t).strip().lower()`, dedupes via a `seen` set, and caps at `tags[:6]`. `_tag_rule` (L57-65) asks for "2-4 short lowercase … tags … free-form, coin apt ones; common examples: identifier, key, metric, …" — explicitly seeded examples, NOT a constrained enum; honors D53/Q2 ("not a locked taxonomy"). `test_draft_extracts_and_normalizes_tags` asserts `["Fact","fact","  Revenue "] → ["fact","revenue"]` and `["Identifier","KEY"] → ["identifier","key"]`. PASS.

3. **LibraryProvider (context.py L177-220).** Conforms to `ContextProvider` (`name = "comment_library"` + `gather(table, columns) -> list[ContextSnippet]`). Does a table-scope lookup (`self._lookup("table", [table])`) and a column-scope lookup (`self._lookup("column", cols)`). Snippet text is **suggestion-only**: "REUSE this wording if it fits this table's evidence; adapt or ignore it if the data differs (do not copy blindly)" (`_fmt`, L218-220) — the D52/Q1 guardrail; never an instruction to copy verbatim. Takes an **injected** `LibraryLookup = Callable[[str, Sequence[str]], list[dict]]` (L174); imports nothing infra. Errors degrade through `ContextGatherer.gather`'s per-provider try/except (L324-328). The provider never mutates drafts / never auto-copies — it only emits context snippets; the sunset exclusion lives in U103's `list_library_for_reuse` query (correct per the ledger/D52). `test_library_provider_emits_suggestion_only_snippets` / `_degrades_via_gatherer_on_lookup_error` / `_no_matches_returns_empty` cover all three. PASS.

4. **Reasoner reuse rule.** `_REUSE_RULE` (L50-54) — "If the CONTEXT contains a previously-approved canonical comment (source comment_library) … PREFER reusing its wording when it fits the evidence — adapt it … do not copy it blindly if the data differs." Interpolated into BOTH the table system prompt (L166) and the column system prompt (L191). Frames reuse as prefer-when-it-fits (suggestion-only) and names the `comment_library` source. `test_table_and_column_prompts_request_tags_and_library_reuse` asserts `'"tags"'`, `"comment_library"`, and `"prefer reusing"` are all present in the combined system prompts. PASS.
