## Checks

### 1. mount correct — PASS
- `api.py:248-249`: `if static_dir and os.path.isdir(static_dir): app.mount('/', StaticFiles(directory=static_dir, html=True), name='spa')`. `html=True` is set, mount path is `/`, and the mount is the **last** statement in `create_app` before `return app` — registered after `/health`, `/api/config`, `/api/run`, `/api/sessions/{id}`, `/api/sessions/{id}/answers`, `.../review`, `.../apply`. Starlette resolves routes in registration order, so the API + health routes take precedence (matches LLD-frontend §4: "FastAPI mounts `app/static` at `/`; API under `/api/*`").
- **Guarded**: the `static_dir and os.path.isdir(static_dir)` predicate means a missing/None dir skips the mount, so the app still builds without a `vite build` — verified by the import check and tests 2/3.
- `app/main.py:19-20`: `_STATIC = os.path.join(os.path.dirname(__file__), 'static')` then `create_app(build_service(...), static_dir=_STATIC)` — passes `app/static` as required.
- Mutation probe confirmed `app.routes[-1]` is the `''`-path Mount (the `/` SPA mount), after all `/api/*` and `/health` paths.

### 2. test meaningfulness — PASS
`tests/test_static_mount.py` (3 tests) covers exactly the three promised scenarios:
- `test_spa_served_at_root_and_api_takes_precedence`: writes `index.html`, asserts `GET /` → 200 with body content AND `GET /api/config` → 200 AND `GET /health` → `{'status':'ok'}`. This positively guards API precedence — if the mount were registered *before* the API routes, the `/` Mount would shadow `/api/config` (SPA fallback / 404) and the assertion would fail. (Confirmed by the route-order probe: precedence is order-dependent and the test would catch a reordering regression.)
- `test_no_mount_without_static_dir`: no `static_dir` → `GET /` → 404, `/health` still 200 (no-dir → no mount).
- `test_missing_static_dir_is_ignored_not_crash`: `static_dir='/no/such/dir'` → app builds, `GET /` → 404, `/health` → 200 (missing-dir → no crash; this is the `os.path.isdir` guard's mutation probe — dropping the guard would raise at `StaticFiles(directory=...)` construction and crash app build).
The mutation-probe coverage is genuine: each of the three asserted behaviors fails under the obvious corresponding mutation (reorder mount, drop guard, mount unconditionally).

### LOW severity
- **SPA fallback for client routing is implicit, not asserted** (`api.py:249`, LLD-frontend §4 + §6: "SPA fallback route serves `index.html` for client routing"). `StaticFiles(html=True)` does serve `index.html` for `/` and for directory paths, but a deep client-route GET (e.g. `/review/abc`) for a path with no matching file returns 404 under `html=True` rather than falling back to `index.html`. For the demo's light client router this may matter on hard-refresh of a sub-route; not exercised by any test. Defensible for v1 (the LLD calls the router "light" and the app entry is `/`), noting only as a deviation from the literal §4 "SPA fallback route" wording. Non-blocking.
