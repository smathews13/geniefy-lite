## Detailed checks (all PASS unless noted)

**Migration 004 coherence — PASS**
- `set search_path to geniefy;` present (`004:8`); every object is `geniefy.`-qualified (`schema_runs`, `schema_run_status`, the `sessions` ALTER, indexes, trigger). Matches 001 house style.
- `schema_runs.id uuid primary key default gen_random_uuid()` (`004:16`) — satisfies U101 audit **M3** (uuid PK, not the LLD's draft `TEXT`). `gen_random_uuid()` is available because 001 does `create extension if not exists pgcrypto` (`001:7`). FK `sessions.schema_run_id uuid references geniefy.schema_runs(id)` is uuid→uuid (type-consistent).
- `schema_run_status` enum created via `do $$ … exception when duplicate_object then null; end $$` guard (`004:10-13`) — idempotent, matches 001's enum pattern. Values `enumerating|running|completed|completed_with_errors|failed|cancelled` ⊇ the LLD §4 set (adds `cancelled`, which the cancel route D51/§5 needs — coherent superset).
- `create table if not exists` (`004:15`); `alter table … add column if not exists schema_run_id` (`004:31-32`); both indexes `if not exists` (`004:28,33`). Re-runnable.
- Trigger is drop-then-create inside a `do $$` block and reuses `geniefy.set_updated_at()` from 001 (`004:36-40`) — idempotent.
- References only columns that exist: the ALTER targets `geniefy.sessions` (defined `001:99`); the trigger function is defined `001:66`. No grants/revokes (confirmed by grep).

**No collision with 003 — PASS**
- 003 only does `alter type geniefy.session_mode add value if not exists 'hands_off'` — disjoint from 004 (which never touches `session_mode` or its enum). Numbering sequential (001→002→003→004), each file touches disjoint objects. The 004 header comment correctly notes the `'hands_off'` value is added in 003.

**save() linkage — PASS**
- `schema_run_id` is the LAST column in the INSERT list and the LAST value (`store.py:123-124,136`), so all pre-existing positional param indices (`params[0..10]`) are unchanged — confirmed by the still-passing `test_save_writes_session_snapshot_and_normalized_fields` (asserts `params[10] == created_by`).
- `schema_run_id` is **NOT** in the `ON CONFLICT (id) DO UPDATE SET` clause (`store.py:125-131`) — written on first INSERT, preserved across progressive updates. Asserted directly by `test_save_writes_schema_run_id_on_insert_only` (`schema_run_id = EXCLUDED` not in SQL).

- Default `schema_run_id=None` on both `save` and `_write` (`store.py:92,117`) → existing non-hands-off callers unaffected (the whole prior test corpus still passes).

Both paths tested (`test_bump_counts_increments_known_key_with_bound_params` asserts `params == (key, key, run_id)`; `test_bump_counts_rejects_unknown_key`).
- `session_exists` splits `table_ref` via `_split_target` and binds `(run_id, cat, sch, tbl)` (`store.py:482-493`); tested true/false + param binding (`test_session_exists_binds_run_and_split_table`).

**list_sessions filter — PASS**

**SQL-injection posture — PASS**
- Mechanical sweep of every f-string interpolation: every `{…}` is `{s}` (= `self._schema`, allowlisted via `_qualify`'s `_IDENT` regex), with two exceptions — `{bump}` (a fixed literal `"usage_count = usage_count + 1, "` or `""`, no user input) and `', '.join(sets)` in `update_schema_run` (where `sets` is built only from hard-coded column fragments). The bump key, all run ids, filters, and counts keys are bound params. No value is ever interpolated.

**Hermeticity — PASS**
- `store.py` imports no driver at module load (only `json`, `re`, `uuid`, `contextlib`, `typing`, and `geniefy_core.state`). The connection is injected. Confirmed by grep (no `psycopg`).
