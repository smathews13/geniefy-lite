### LOW severity
- **Numeric `0` survives the noise filter** (`reasoner.py:197-199`): the extraction keeps `int`/`float` values (`isinstance(v, (str, int, float))`) and `str(0).strip()` is `"0"` (truthy), so a fact value of literal `0` would render as the chip text `"0"`. Benign ‚Äî all five facts (owner/freshness/grain/keys/sensitivity) are text fields per `_FACTS_RULE`'s instructions, so the model has no reason to emit a numeric `0` for any of them; and `int`/`float` acceptance is otherwise harmless. Cosmetic, not worth a fix unit.
