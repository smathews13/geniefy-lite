## Checks

### Check 1 — each LOW closed correctly — PASS

**LOW-A (U59): `paused`/`cancelled` route to no branch.** CLOSED.
- `App.tsx:91-96` adds a catch-all branch: `session && !IN_FLIGHT.includes(status) && !DONE.includes(status) && status !== 'failed'` → renders a fallback card ("This run is *{status}*. Start a new run to continue.").
- `SessionStatus` (`types.ts:4-7`) has exactly 11 values; IN_FLIGHT (5) + DONE (3) + `failed` (1) = 9, leaving precisely `paused` and `cancelled` — both captured by the catch-all. No status now falls through to a blank `<main>`.
- **No shadowing of IN_FLIGHT/DONE/failed:** the catch-all's guard explicitly negates all three prior predicates, so it is mutually exclusive with them. The four branches partition the status space with no overlap. Correct.

**LOW-B (U60): apply errors not surfaced.** CLOSED.
- `ReviewView.tsx:49-53`: `{apply.isError && (<p …>Apply failed — nothing was written to Unity Catalog. Check warehouse access (MODIFY) and retry.</p>)}`. `useApply` exposes `isError`; the banner renders a whole-request failure (the gap U60 flagged). Per-item `apply_status` badges in DraftCard are unaffected. Correct.

**LOW-C (U60): edit-text stale after a resume re-draft.** CLOSED, correctly guarded.
- `DraftCard.tsx:52-54`: `useEffect(() => { if (!editing) setText(draft.proposed_comment ?? '') }, [draft.proposed_comment, editing])`. Syncs the edit buffer when the server-side proposed comment changes, and the `!editing` guard prevents clobbering an in-progress edit. Dependency array `[draft.proposed_comment, editing]` is correct and complete; the `setEditing(false)` in `act`'s onSuccess (`:57`) then lets the effect re-seed `text` from the freshly-saved value on the next render. No infinite-loop risk (effect only calls `setText` when not editing). Correct.

**LOW-D (U59): ~570 kB recharts single chunk.** CLOSED (see Check 2 for build evidence).
- `DraftCard.tsx:8`: `const JudgeRadar = lazy(() => import('../viz/JudgeRadar').then((m) => ({ default: m.JudgeRadar })))`. `JudgeRadar` is a **named** export (`viz/JudgeRadar.tsx:5`), so the `.then(m => ({ default: m.JudgeRadar }))` default-shim is the correct adapter for `React.lazy` (which requires a default export). Rendered inside `<Suspense fallback={<div className="h-40 animate-pulse …" />}>` at `DraftCard.tsx:163-165`. Note: `viz/index.ts:5` still statically re-exports `JudgeRadar`, but DraftCard imports it only via the direct lazy path (its barrel import at `:4` is `ConfidenceChip` only) — and the build confirms Rollup tree-shook the static barrel re-export so recharts does **not** leak into the main chunk.

**LOW-E (U61): SPA deep-route fallback.** Correctly N/A. Grep for `react-router | createBrowserRouter | useNavigate | useRoutes | history.push | window.location.(path|href|assign) | <Route | <BrowserRouter | RouterProvider` over `src/` → **no matches**; `package.json` has no router dependency. Routing is purely state-driven (`App.tsx` switches on `session.status`, single entry at `/`). There are genuinely no client-side deep routes to 404 into, so dropping this LOW is correct.

**Regression / D23.** No regression of the restrained confidence encoding. `ConfidenceChip` (color + icon + numeric + label) is untouched; the diff/edit/approve/reject/"why" surface in DraftCard is intact (edit textarea `:81-109`, approve/reject `:113-128`, why-toggle `:138-146`, explainability `:153-176`). Compliant with D23 ("restrained, not overboard").
