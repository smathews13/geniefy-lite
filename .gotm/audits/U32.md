### LOW severity
- **Declared `TransportError` is dead code.** `src/geniefy_core/llm.py:127-128` defines a `TransportError(Exception)` and the `ChatTransport` docstring (`:110-112`) implies the transport "raises these," but `complete()` retries on a broad `except Exception` (`:194`), never referencing `TransportError`. The behavior is correct (any non-`LLMError` is treated as transient), but the named class is unused — either wire it (catch `TransportError` specifically) or drop it to avoid implying a contract the client doesn't enforce. Non-blocking.
- **`HeuristicTokenCounter` boundary comment is slightly loose.** Test `test_counter_proportional_and_empty` asserts `count("a"*41) >= 10` (round(41/4)=10). Correct, but `round()` uses banker's rounding (e.g. `round(2.5)=2`); fine for a *bound*-cost heuristic and explicitly documented as "good enough to bound cost" (`:40-41`), so not a defect — noted only so a future tightening swaps in a real tokenizer at the app layer as the docstring already says.
