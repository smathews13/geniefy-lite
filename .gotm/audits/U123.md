## Findings

### HIGH severity
- none.

### MEDIUM severity
- **`config.lakebase_endpoint` fallback is non-functional — the field does not exist on `AppConfig`.**
  `api.py:647` reads `os.environ.get("GENIEFY_LAKEBASE_ENDPOINT") or getattr(config, "lakebase_endpoint", None)`.
  `AppConfig` (`src/geniefy_app/config.py:29-41`) defines no `lakebase_endpoint` field and `from_env` never reads
  `GENIEFY_LAKEBASE_ENDPOINT`, so the `getattr(...)` second operand is *always* `None`. The only working endpoint
  source on a cluster is the `GENIEFY_LAKEBASE_ENDPOINT` env var (which the entrypoint always sets — so the live
  hands-off path is fine, and the App path never reaches the mint branch). The defect: the spec's stated
  `config.lakebase_endpoint` fallback silently does nothing, so any cluster caller that did NOT go through
  `schema_run_entry.py` (and relies on the config object) would hit the endpoint-required `RuntimeError` rather
  than the config default. Not a live blocker (env covers the one real cluster path), hence MEDIUM not HIGH.
  - Suggested fix unit: add a `lakebase_endpoint: str | None = None` field to `AppConfig` populated from
    `GENIEFY_LAKEBASE_ENDPOINT` in `from_env`, so the `getattr` fallback is real (or drop the dead fallback +
    its comment and rely on the env var only).

### LOW severity
- **`workspace_bearer` older-SDK dict-arg branch (`except TypeError`, `providers.py:96-99`) is untested.**
  `test_workspace_bearer_falls_back_to_authenticate_on_cluster` only exercises the newer dict-return shape; the
  `_FakeConfig.authenticate()` takes no arg, so the `TypeError`/passed-in-dict path never executes in the suite.
  Logic reads correct, but it is asserted only by inspection.
  - Suggested fix unit: add a fake whose `authenticate()` requires a dict arg, asserting the stripped token.
- **`lakebase_db_token` "no token returned" `RuntimeError` branch (`providers.py:122-123`) is untested.** The
  mint-success and endpoint-required cases are covered; the empty/`{}` response on a cluster is not.
  - Suggested fix unit: assert `RuntimeError` when the credentials API returns `{}` with a non-None endpoint.
