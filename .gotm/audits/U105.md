### LOW severity

- **Approve/apply `match_key` normalization can diverge for non-canonical FQNs.** `apply.py:160-174` derives the table `match_key` from `_split(state.target)` (`apply.py:45-46`), which strips backticks and surrounding whitespace per `.`-segment ‚Üí `f"{cat}.{sch}.{tbl}"`. The write-on-approve path (`api.py:182-188`) uses the raw `state.target` verbatim as the table `match_key`. For a normal FQN (`samples.tpch.orders`) the two are byte-identical and hit the same `(scope, match_key)` row ‚Äî so the approve‚Üíapply upgrade lands on the same entry (verified by `test_review_approve_table_writes_library_as_approved` asserting `samples.tpch.orders` and `test_apply_records_applied_comment_in_library` asserting the same). Only a backtick-quoted or whitespace-padded `state.target` would split them, producing two rows instead of an upgrade. Cosmetic/edge-only; FQNs in practice are unquoted. Could normalize the approve-side key with the same `_split` helper if desired.

- **Empty-comment skip in write-on-approve is not directly unit-tested.** `_write_library_on_approve` correctly early-returns when `proposed_comment` is empty (`api.py:177-179`), but no test exercises that branch in isolation (the `edit` route already 400s on empty before reaching it, so the live exposure is the `approve`-of-an-empty-draft case). Code path is correct by inspection; test gap only.

- **The 501 getattr-guard path (store lacking lifecycle methods) is not unit-tested.** `sunset_library`/`revive_library` raise `SessionError(501, ‚Ä¶)` when the store lacks the method (`api.py:194-203`), and `_write_library_on_approve`/`_write_library` silently skip when `upsert_library_entry` is absent ‚Äî but the test fakes (`tests/test_api.py:70-77`, `tests/test_apply.py:79-80`) all implement the methods, so the absent-method branches are reachable-by-inspection only, not asserted. Low risk; the guard is a defensive no-op for older fakes.
