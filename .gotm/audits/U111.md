### LOW severity
- **Stale docstring in `_make_run_schema_job` (`api.py:651`).** The docstring says "The Job (U111) reads `task=schema_run` and drives enumerationтАж", but the trigger correctly passes **no** `task` param (that was the very bug U111 fixes) тАФ the two-task DAB design routes by task, not by a `task` job-parameter. The code is correct; only the comment is misleading residue from the pre-fix design. Cosmetic.
- **`run_schema` `created_by` default `"app-service-principal"` (`schema_run.py:24`) is a free-string sentinel.** The driver is invoked by the entrypoint without passing `created_by`, so every per-table session is attributed to this literal SP marker. This is correct per D48 (batch body runs as SP; human attribution lives on the `schema_runs.created_by` set at trigger time, U110), but the string is not derived from the actual SP identity. Acceptable for v1; noted only as a cosmetic precision nit.
