### 4. D18 poll loop — PASS
`useSession` (`hooks.ts:35-45`) sets `refetchInterval` to a function that returns `POLL_MS` (1500) while `status ∈ IN_FLIGHT` and `false` otherwise. `IN_FLIGHT = [created, profiling, gathering_context, reasoning, applying]` matches LLD §5's poll-while set exactly. Polling stops on every terminal status (`awaiting_input`, `ready_for_review`, `applied`, `paused`, `failed`, `cancelled`). The frontend's in-flight set is a superset of the backend's `_IN_FLIGHT` by `created` only — correct, since `created` is a transient pre-run status the client should keep polling through. Mutations (`useAnswers`/`useReview`/`useApply`) invalidate `['session', id]` on success, re-arming the poll. ✓

### 5. D23 fidelity (confidence = color + icon + label, P3) — PASS
`ConfidenceChip.tsx` renders **color** (band color as text + tinted background), **icon/shape** (`BAND_ICON` ○/◐/● — explicitly "shape also encodes the band, so the signal survives without color", `confidence.ts:25`), and **label** (`%` value visibly + a `sr-only` "{High/Medium/Low} confidence" for screen readers + a `title`). Color is never the sole signal. Bands are consistent: `band()` thresholds (≥0.8 high, ≥0.6 mid, else low) and `BAND_COLOR` align with the Tailwind `confidence.{low:#64748b, mid:#f59e0b, high:#16a34a}` scale and the D23 cool-grey → amber → green ordering. ✓ The other primitives (ReadinessMeter, NullFractionBar, Sparkline, JudgeRadar) all carry `title`/`aria-label`/text labels, consistent with the accessibility guardrail.
