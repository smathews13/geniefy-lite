### MEDIUM severity
- **The ambiguous-trip tests under-verify two of its three required sub-conditions (test-meaningfulness gap, U31).** U4 §3.6 requires the first force-low rule to fire **only** when *ambiguous name* AND *no usage evidence* AND *high null fraction* are all true. Mutation testing of `gate.py:78` shows the suite catches removal of the `not has_usage_evidence` sub-condition (via `test_ambiguous_but_used_does_not_trip`, `test_gate.py:89`) but does **NOT** catch: (a) dropping `signals.high_null_fraction` (Mutation 3 — 15/15 still passed), nor (b) dropping `signals.ambiguous_name` (Mutation 9 — 15/15 still passed). The suite tests the all-true positive and exactly one breaking negative (usage evidence); it never asserts that `high_null_fraction=False` alone or `ambiguous_name=False` alone breaks the trip. A future edit that weakened this rule along those two axes would route confident-but-unused/null columns to `keep` undetected. The **module code is correct** — this is a coverage gap in the tests, not a defect in `gate.py`.
  - Suggested fix unit: append a Gate-test-hardening unit adding two cases — `HardSignals(ambiguous_name=True, has_usage_evidence=False, high_null_fraction=False)` and `HardSignals(ambiguous_name=False, has_usage_evidence=False, high_null_fraction=True)` — each asserting `.keep` (no trip). Mirrors the U29 spine-test-hardening follow-on.

### LOW severity
- **No standalone unit test for the `None`-target table draft in `apply()`.** `apply()` keys the table draft as `None` (`gate.py:114`) and `_signals()` provides `None: HardSignals()`, but the table draft in `_state()` is constructed to `keep` (confidence 0.9, no signals), so the `None`-keyed *force-low / needs_input* branch for a table is never exercised end-to-end. `test_question_text_variants` (`test_gate.py:164`) does cover the `DraftKind.TABLE`/`target_name=None` question-text path directly, which mitigates this. Low impact; fix if convenient by adding a table-draft-trips case.

## Detailed check results

**2. Spec match (U4 §3.6) — PASS.**
- *Keep rule:* `keep` iff `confidence >= keep_threshold` AND no hard-signal trip; else `needs_input` (`route_item`, `gate.py:86-96`). The comparison is `if confidence < keep_threshold: needs_input`, i.e. the boundary is **inclusive** (`confidence == threshold` keeps) — a deliberate choice, asserted by `test_threshold_is_inclusive` (`test_gate.py:50`) and consistent with the spec's `overall ≥ keep_threshold`. Mutation 1 (`<`→`<=`) and Mutation 2 (`<`→`>`) were both caught.
- *Three force-low trips* (`hard_trips`, `gate.py:75-84`) match §3.6 exactly: (a) `ambiguous_name and not has_usage_evidence and high_null_fraction`; (b) `enum_candidate and not has_decode_source`; (c) `keylike and not has_key_target`. Each requires ALL its sub-conditions. Trips override the score regardless of confidence (checked before the threshold; `gate.py:90-91`) — verified by `test_*_forces_low_even_if_confident` at confidence 0.99/0.95. (See the MEDIUM finding re: test coverage of trip (a)'s sub-conditions.)
- *Interactive vs batch:* interactive → set `NEEDS_INPUT` + targeted `Question`s returned (`gate.py:126-135`); batch/hands-free → set `LOW_CONFIDENCE` + reason appended to rationale, asks nothing, `open_questions=[]` (`gate.py:136-140`). Matches §3.6. Mutation 7 (mode predicate inverted) was caught. Question text is targeted per reason token (`question_text`, `gate.py:144-153`), e.g. the `tier`/enum example mirrors the §3.6 sample.
