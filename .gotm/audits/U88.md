## Checks

### 1. Regenerate wiring — PASS
`useRegenerate` (hooks.ts:105-112) is built on `useSessionMutation` (hooks.ts:72-81), reusing the
exact pattern shared by `useReview`/`useAnswers`/`useApply`. It POSTs to
`/api/sessions/${sessionId}/regenerate` with `JSON.stringify(body)` where `body: {targets?: string[]; all?: boolean}`,
and on success `onSuccess` invalidates `['session', sessionId]` — so the `useSession`
poll (hooks.ts:60-70) re-fetches and surfaces the re-draft. The per-draft button
(DraftCard.tsx:147) sends `{ targets: [target] }`; for the table card `target` is `TABLE_TARGET`
= `'__table__'` (types.ts:132; passed in ReviewView.tsx:27), for columns it is the column name
(ReviewView.tsx:30-31) — both exactly match U83's accepted targets (`"__table__"` / column names).
The "Regenerate all" button (ReviewView.tsx:50) sends `{ all: true }`. Body shape and endpoint
match D45 / §E3 / U83 verbatim.

### 2. Async/poll consistency (D18) — PASS
The mutation fires-and-invalidates only — no `await`/blocking that would stall the loop.
After 202, U83 saves an in-flight status, `invalidateQueries(['session', id])` triggers a refetch,
and `useSession`'s `refetchInterval` resumes polling because the status is in `IN_FLIGHT`
(hooks.ts:23-26,65-68). Same mechanism the existing run/resume path uses; no regression.

### 3. Delight restrained (D23) — PASS
Only Tailwind utility transitions are used: card lift `transition-shadow duration-200 hover:shadow-md`
(DraftCard.tsx:65) and button `transition hover:bg-*` (DraftCard.tsx:140,149; ReviewView.tsx:52).
No new keyframes/custom animation in any changed file (grep for `keyframes|animation-|will-change`
returned none). `src/index.css` is unchanged — still only the three `@tailwind` directives
(mtime 2026-06-12, predates U88's 2026-06-13 work); `tailwind.config.js` likewise unchanged
(mtime 2026-06-12). `package.json` deps are identical to the U58 set (recharts/visx/tanstack/clsx/
tailwind-merge) — **no new npm dependency**. Honors D22/D23 "restrained, not overboard."
