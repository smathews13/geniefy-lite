## Detailed check results

**Check 1 — the U62 LOW is closed: PASS.**
The `needval` helper (`deploy.sh:38`) guards each valued flag: `needval() { [[ -n "${2:-}" ]] || {
echo "option $1 requires a value" >&2; usage 1; }; }`. The `${2:-}` default-expansion means `set -u`
no longer fires on a missing value, so there is no raw `$2: unbound variable`. The three case arms
(`-t`/`-p`/`--host`, lines 41–43) each call `needval "$1" "${2:-}"` before dereferencing `$2`. Live
runs of `-t`, `-p`, and `--host` with no value each print the friendly `option <x> requires a value`
message + usage and exit non-zero (1). The raw bash error is gone.

**Check 2 — valid parsing still works + no early side effects: PASS.**
A valid `-t dev` (followed by `-h`) parses and exits 0 — the guard only triggers on a *missing*
value (`[[ -n "${2:-}" ]]` is satisfied when a value is present). Combined valid flags
(`-t dev -p myprof --host https://x -h`) also exit 0. The guard lives entirely inside the arg-parse
`while` loop (lines 39–49), which runs before `ROOT="$(...)"` / `cd "$ROOT"` (lines 51–52) and before
every `step` block (steps 1–7 begin at line 70 onward). No deploy side effect can occur before the
guard runs. `usage 1` routes through the existing `usage()` function (exit code arg), so the message
+ full usage text print together.

The only changes from the U62-audited file are the additions the fix unit promised: the single
`needval` helper line (line 38) and the in-line `needval "$1" "${2:-}";` prefix on the three valued
case arms (lines 41–43). Structural evidence that nothing downstream shifted: the U62 audit cited the
seven `step` headers at lines 69/82/92/110/116/130/142 and the U62 audit's own line references
