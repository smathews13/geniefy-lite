### MEDIUM severity
- **Minimum-version floor (`v < floor`) is unexercised.** `profiler.py:612-616` rejects an in-major-but-below-minimum version (`unsupported`, "… < minimum …"). No test sets a sub-floor same-major version, and `min_schema_version` is never passed to the `Profiler` ctor in any test (`grep min_schema_version tests/test_profiler.py` → empty). Mutation MUT7 (`if v < floor` → `if False`) leaves all 33 green, confirming the branch is dead to the suite. The version gate is a named U18 deliverable and a real U3 §4.2 / U4 §3.2 contract clause, so the floor half of it is materially uncovered. (With the current defaults `PROFILE_SCHEMA_VERSION == MIN_SUPPORTED_PROFILE_VERSION == "1.0"` the branch is unreachable via defaults — it only fires when a caller raises `min_schema_version` above the core target — so impact today is bounded; the gap is in *test* coverage of the gate, not a module defect.)
  - Suggested fix unit: add a test passing `Profiler(..., min_schema_version="1.5")` against a provider returning `"1.2"` and assert `ProfileError.code == "unsupported"`.

### LOW severity
- **`information_schema_queries` "constraints" body is unchecked.** `test_information_schema_queries_qualified_and_escaped` (tests:291-295) asserts the three query keys exist and spot-checks the `columns` query's qualification/escaping, but never inspects the `constraints` query string (the PK/FK `table_constraints`/`key_column_usage` join — `profiler.py:407-417`). A regression in that join would pass. The escaping test (tests:298-301) does cover the SQL-injection concern broadly.
  - Suggested fix unit: assert `key_column_usage` / `table_constraints` appear in `q["constraints"]` and that the FK join references `constraint_column_usage`.
- **`ProfileOptions` non-default fields not asserted in request shaping.** `test_request_to_dict_shape` (tests:100-105) checks `options.top_k` but not `include_samples`, `max_cardinality_for_topk`, or `reuse_analyze_stats` (`profiler.py:137-143`, all U3 §4.1 fields). A drop of any of those three from `to_dict` would not be caught.
  - Suggested fix unit: extend the request-shape assertion to cover all four option keys.
- **`SampleSpec("percent")` value-range boundaries (0 and 100) untested.** tests:122-127 confirm `150` is rejected, but the inclusive `0 <= value <= 100` boundary (`profiler.py:118`) isn't pinned at the edges.
