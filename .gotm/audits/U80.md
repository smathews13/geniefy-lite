### MEDIUM severity
- **Jitter test does not distinguish full-jitter from plain backoff.** `test_backoff_is_full_jitter_within_bounds` (test_llm.py:158-164) asserts only `all(0.0 <= s <= base * 2**i ...)`. The *prior* plain backoff `base * 2**attempt` produces exactly the upper bound (`[2.0, 4.0, 8.0]` for base=2), which still satisfies `s <= base*2**i`. Verified: the same assertion passes against simulated prior plain-backoff sleeps. So this test would NOT fail against the old (no-jitter) implementation — it is a weak "bite." The production code *does* implement full jitter correctly (llm.py:238), so this is a test-strength gap, not a behavior defect. A stronger test would assert variance across runs (e.g. seed `random` and assert a value strictly inside the open interval, or assert not-all-equal-to-upper-bound).
  - Suggested follow-on unit: strengthen `test_backoff_is_full_jitter_within_bounds` to assert at least one sleep is strictly below its upper bound (or seed `random` and assert exact jittered values).
