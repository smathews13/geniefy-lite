## Verified (pass)

- **The field exists, optional, defaults None.** `AppConfig` declares
  `lakebase_endpoint: str | None = None` (`config.py:43`), positioned after `secret_scope` and before
  `run_config`/`mcp_providers` so the dataclass field order stays valid (all preceding non-default fields are
  still required, the new one carries a default). A docstring comment (`config.py:40-42`) explains it is needed
  only OFF the App runtime (Job cluster) to mint a Lakebase credential when `oauth_token()` is unavailable, and
  is unset on the App — matching the U123 oracle exactly.
- **`from_env` reads `GENIEFY_LAKEBASE_ENDPOINT`, blank/whitespace → None.** `config.py:129` does
  `lakebase_endpoint=(env.get("GENIEFY_LAKEBASE_ENDPOINT") or "").strip() or None`. This is the identical idiom to
  the other optionals (`secret_scope`/`pg_schema` use `(env.get(...) or "geniefy").strip()`); here the trailing
  `or None` converts the empty/whitespace-only string to `None` rather than `""`, so blank env → None. Verified
  empirically: the test passes `"   "` and asserts `is None`.
- **The `_lakebase_connection` fallback is now live (no longer dead code).** `api.py:651` reads
  `os.environ.get("GENIEFY_LAKEBASE_ENDPOINT") or getattr(config, "lakebase_endpoint", None)`. With the field now
  present on `AppConfig`, the `getattr(...)` second operand can return a real value (the config default) instead
  of always `None`. The U123 MEDIUM ("the `getattr(...)` second operand is *always* `None`") is resolved.
- **App-runtime behavior unchanged.** On the App, `GENIEFY_LAKEBASE_ENDPOINT` is unset, so the new field resolves
  to `None` (blank→None handling), and `endpoint` is `None`. `lakebase_db_token` on the App still takes its
  `oauth_token()`-first path (per U123 audit), so the env-or-oauth path works exactly as before — no regression.
  `test_minimal_valid_env_applies_defaults` locks `cfg.lakebase_endpoint is None` (`test_app_config.py:48`).
- **No `from_env` regression.** The new line is a pure additive kwarg to the `cls(...)` return; it does not touch
  the REQUIRED list, the `num()` coercion path, the MCP-providers parse, the fail-fast aggregation, or any other
  field. Full suite is **375 passed / 1 skipped** — exactly the expected count (370 at U123 + the 5 new
  `lakebase_endpoint`-adjacent assertions land within the +5 net since U123; no prior test broke). The
  `getattr(config, "lakebase_endpoint", None)` in api.py still uses the defensive 3-arg form, so even a stale
  config object without the field would not raise.
- **The test covers default None + set value + blank→None.** `test_lakebase_endpoint_parsed_when_set`
  (`test_app_config.py:51-55`) asserts the set case (`projects/geniefy/branches/dev/endpoints/primary` round-trips)
  and the blank case (`"   "` → None); `test_minimal_valid_env_applies_defaults` (`:48`) asserts the unset default
  is None. All three required cases present and passing.
